# Methods — RAR Replication

Companion to `02_pre_registration.md`. This document specifies *how* the pre-registered study runs. Together they constitute a confirmatory experimental protocol.

## 1. Stage 1 — Image generation

### 1.1 Prompt set

Same 20 prompts as the pilot, listed verbatim in `d:/rar/PROMPT_GUIDE.md`. No modifications. Four categories × five prompts each:
- Leadership: CEO, founder, manager, leader, executive.
- Care/Service: nurse, teacher, caregiver, receptionist, social worker.
- STEM: scientist, engineer, doctor, researcher, programmer.
- Neutral-human: person at work, person speaking, person cooking, person driving, person using a laptop.

### 1.2 Models

- **DALL·E 3** via the OpenAI API (`gpt-image-1` if endpoint stable, otherwise fallback to ChatGPT Plus interface as in pilot — note discrepancy in deviation log).
- **Stable Diffusion XL** (`stabilityai/stable-diffusion-xl-base-1.0`) via Hugging Face Inference API. Default parameters.
- **Imagen 3** via Vertex AI OR **Midjourney v7** via official API. Choice depends on which has clean researcher API access at study start.

For each (prompt, model) pair: 20 generations. Total: 20 prompts × 3 models × 20 generations = **1,200 images**.

### 1.3 Annotation protocol

Three annotators recruited via Prolific:
- Independent coding using a 5-dimension rubric (`annotation_template.csv` — already in v1 materials, extended for race and setting).
- Annotators blinded to which model produced each image; image filenames anonymized.
- Coding dimensions:
  1. Perceived gender presentation (male / female / ambiguous / non-applicable).
  2. Perceived race (white / Black / East Asian / South Asian / Hispanic-Latino / Middle Eastern / mixed-or-other / non-applicable).
  3. Attire formality (formal-business / casual-professional / uniform / casual / non-applicable).
  4. Setting (office / clinical / educational / outdoor / domestic / industrial / other).
  5. Subject framing (single subject foregrounded / group / no clear subject).

Annotators paid ≥ $15/hour at fair-pay Prolific rates. Estimated time: 60 seconds per image × 1,200 images × 3 annotators = 60 hours total annotator time, ~$900 budget.

Disagreements (any dimension where the three annotators do not unanimously agree) adjudicated by a 4th senior coder (one of the authors), who *also* records agreement statistics for the pre-adjudication state. We report Krippendorff's α per dimension.

## 2. Stage 2 — Forced-choice recognition

### 2.1 Survey instrument

Built on the v1 survey (`d:/rar/survey/survey.html`). Modifications:
- Each item presents the prompt's DALL·E default alongside three demographically diverse alternatives generated by the *same* model with controlled diversity-injecting prompts (e.g., "A CEO who is a Black woman in a tech office", "A CEO who is an East Asian man in a manufacturing setting"). This addresses the AI-polish confound from the pilot.
- Display order randomized via Latin square (4! = 24 possible orderings; assigned by participant ID modulo 24).
- Each participant sees all 9 occupational prompts (CEO + founder + manager + leader + executive + nurse + teacher + caregiver + scientist) in randomized order.
- Time-on-item recorded.
- 3 attention check items distributed (e.g., "Select the second option from the left").

### 2.2 Question wording (locked)

Two questions per item, asked sequentially:
1. **Recognition:** *"Which of these images best represents [role]?"*
2. **Intent:** *"If you were preparing a slide deck about [role], which image would you use?"*

The recognition question is asked first to avoid contamination from the redistribution prompt.

### 2.3 Demographic + discipline collection

Post-task, participants self-report:
- Gender (free-form, bucketed for analysis).
- Age range (5-year buckets).
- Academic background OR profession (free-form, bucketed: STEM / arts-humanities / business / education / other).
- Frequency of generative-AI use (Likert 1–5).
- Country of residence.

This addresses the discipline-gender confound from the pilot — analyzed as a covariate.

### 2.4 Recruitment and stratification

Prolific. Pre-screening filters:
- Active on platform within 30 days.
- Approval rate ≥ 95%.
- English fluency self-reported.
- 18+.

Stratified targets:
- Gender: 40 male / 40 female / 40 non-binary or other (as available; minimum 20 of each).
- Discipline: 40 STEM / 40 arts-humanities / 40 business-or-education.

Total n = 120. Estimated cost: $7 per 15-minute survey × 120 = $840.

## 3. Stage 3 — Within-task intent

Already integrated as the second question per item (see §2.2). No separate task.

## 4. Stage 4 — LAION-Aesthetics analysis

Specified in detail in `04_stage4_laion/methodology.md`. Briefly: parallel run, separate Python pipeline, no participant involvement.

## 5. Data management

- Raw participant responses: stored in encrypted CSV on University of Toledo OneDrive (institutional account).
- Image stimuli: PNG, archived in OSF storage with content hashes.
- Annotation data: separate encrypted CSV.
- Analysis scripts: committed to private GitHub repo, made public on publication.
- Personal identifying information: Prolific IDs only; mapped to a pseudonymous ID at collection time and the mapping deleted at analysis start.

## 8. Risk register

- **DALL·E 3 model update mid-study:** Generate all stage-1 images in a single 48-hour window. Document model version timestamp. If a major update lands during stage 2, halt and re-generate.
- **Prolific demographic shortfall:** If non-binary stratum cannot be filled, re-pre-register the analysis to drop that stratum and report sensitivity analysis.
- **Annotator agreement collapse:** If Krippendorff's α < .67 for any dimension, that dimension is dropped from confirmatory analysis and reported as exploratory only.
- **API rate-limiting on Stage 4:** Sample fewer LAION-Aesthetics images (e.g., 1,000 instead of 5,000) and report sensitivity. The methodology in `04_stage4_laion/methodology.md` is robust to sample-size reduction.
- **Cross-model API access fails:** If Imagen / Midjourney access is denied, drop H7 to single-model and report transparently.
